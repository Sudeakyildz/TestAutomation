Manuel test yavaş, tekrarlı ve hataya açık

Manuel testin sorunları

  • Her release öncesi aynı adımlar elle tekrarlanıyor
  • API çalışıyor ama UI bozuk olabiliyor
  • Hata geç fark ediliyor
  • Sonuçlar ölçülemiyor, kişisel notlarda kalıyor

Hedef

  • Staging'de otomatik, tekrarlanabilir testler
  • Tek komutla smoke veya regression
  • CI ile her push'ta erken uyarı
  • Panelden test başlatılabilsin

Ne inşa ettik?

Üç parçalı kalite güvence sistemi

1 — Test paketi

pytest ile ~170 test: API, E2E, sandbox write, unit. Page Object Model ile bakımı kolay yapı.

2 — Web paneli

Node.js panel: Smoke / Regression / Write butonları, kalıcı sonuçlar.

3 — CI pipeline

GitHub Actions: PR'de smoke, main'de regression. Secrets repoda değil.

Ne kazandık?

  • 5–10 dk'da kritik akışlar (smoke)
  • Regression CI'da ~20–25 dk
  • Write testleri ayrı paket + cleanup
  • Bilinen bug'lar xfail ile işaretlendi
  • Preflight: koşmadan önce ortam kontrolü
Manuel regresyonun büyük kısmını otomatikleştirdik.
